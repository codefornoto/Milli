バイブコーディングとは？

バイブコーディング（Vibe Coding）とは、プログラミングの難しい専門知識がなくても、AI（ChatGPTやGeminiなど）と自然言語で対話しながら、まるで会話のノリ（バイブス）で直感的にアプリやレポートを開発していく手法のことです。

Webアプリを構成する3つの言語

AIに指示を出す前に、Webページが何で出来ているか、ざっくり理解しておきましょう。

HTML

骨組み・構造

文章、画像、リンクなど、ページに「何を表示するか」を定義します。家の建築で言えば「柱や壁」です。

CSS

装飾・デザイン

色、大きさ、配置など、ページを「どう見せるか」を定義します。家の建築で言えば「壁紙や塗装」です。

JavaScript

動き・機能

クリックで開く、データでグラフを描くなど「どう動かすか」を定義します。家の建築で言えば「電気や水道」です。

今回はこれら3つを 1つのHTMLファイル にまとめてAIに作ってもらいます。

Step 1：HTMLだけでページを作る

HTML（HyperText Markup Language）は、Webページの「骨組み・構造」を作る言語です。

Step 3：JavaScriptで動きをつける（機能紹介）

JavaScript（JS）は、Webページに「動き・機能」を追加する言語です。
AIに指示を出すためには、実現したい機能の「一般的な名称（専門用語）」を知っておくことが非常に重要です。

AIへの指示で役立つキーワード

  • ハンバーガーメニュー： スマホ等で三本線（≡）を押すと出てくるメニュー。
  • モーダルウィンドウ： 画面の真ん中にポンッと浮き出て、背景が暗くなる小窓。
  • アコーディオン： 見出しをクリックすると、ビヨーンと詳細が開閉する仕組み。
  • カルーセル（スライダー）： 画像や要素が横にスライドして切り替わる機能。
  • ドラッグ＆ドロップ： マウスで要素を掴んで、別の場所に移動させる操作。

プロンプト（AIへの指示書）を作ろう

AIにイメージ通りのレポートを作ってもらうためには、以下の4つの要素をプロンプトに含めることが重要です。

1. 出力形式

「HTMLベースのレポートを作りたい」と明確に伝えます。

2. データの説明

「石川県の観光アンケートデータです」など、入力するデータがどんなものなのかを説明します。

3. レポートに欲しい機能

「グラフやランキングでまとめて」「自由記述はすべて見られるようにして」など、具体的な要件を書きます。

4. トンマナの指示

「全体的に明るいポップな色を使って」「夏を感じるカラートーンにして」など、デザインの雰囲気を伝えます。

プロンプトを作る準備

プロンプトを書く前に、データの中身と「誰に見せるのか」を整理しておきましょう。

A データを用意する

  • Milliの全データでは処理が重すぎる場合があります。
    EXCELのフィルター機能などで、施設や期間を絞ることをオススメします。（目安：〜数千件くらい）
  • ChatGPTなら、EXCELファイルをそのままアップロードして解析してくれます。
  • うまくいかない時は、EXCELのデータを全選択してコピー＆ペーストでもOKです。

B ターゲットを考える

誰に見てもらう？
社長？ 行政関係者？ 事業者？ 現場のスタッフ？

→ ターゲットによって、必要な見せ方（経営者視点、課題の改善点、評価されている点など）が変わります。プロンプトにも「〇〇に見せる用です」と記載しましょう。

プロンプトの例

先ほどの要素を組み合わせた、実践的なプロンプトの例です。

例1：一里野温泉（社長に見せる用）

データは一里野温泉の観光アンケートデータです。
これらのデータをまずは可視化し、自由記述も確認出来るようなHTMLベースのWebアプリを作って。印刷しても綺麗なように。
このレポートアプリは一里野温泉スキー場の社長さんに見せるから、経営者視点で、改善点・評価されている点なども載せて欲しい。

例2：いしかわ動物園（行政・職員に見せる用）

データはいしかわ動物園のアンケートデータです。
これらのデータをまずは可視化し、自由記述も確認出来るようなHTMLベースのWebアプリを作って。印刷しても綺麗なように。
このレポートアプリは行政関係者、動物園職員に見せるから、施設管理者視点で、改善点・評価されている点なども載せて欲しい。

ハンズオンの手順

  1. AIに投げる用のデータを作る（EXCELでフィルタリング等）
  2. プロンプトを作る（ターゲットを意識、CSSも入れる）
  3. AI（ChatGPTなど）に投げて、コードを生成させる
  4. 出力されたHTMLコードをコピーして、メモ帳に貼り付ける
  5. 「〇〇.html」という名前で保存する（※ファイルの種類は「すべてのファイル」を選ぶ）
  6. 保存したファイルをダブルクリックしてブラウザで開く

作ったレポートを共有・公開する

作成したHTMLファイルは、Googleサイトなどを使えば簡単にチームや外部と共有できます。

⚠️ セキュリティに関する超重要注意事項

  • 機密情報（個人情報など）を含む場合は、絶対に一般公開設定にしないこと！
  • 機密情報を含むHTMLを共有する場合は、Googleドライブの「アクセス制限付き共有フォルダ」のリンクや、HTMLファイルそのものを直接送付して共有すること。
  • 個人情報を含まない「オープンデータ」であれば、外部公開可能です。

Googleサイトでの公開手順

  1. Googleアカウントにログインする。
  2. Googleサイト (sites.google.com) にアクセスする。
  3. 「空白」のサイトを新規作成する。
  4. 右側のメニューから「ページ」タブを選択。
  5. 下部にある「＋」ボタンにカーソルを合わせ、「ページ全体の埋め込み」を選択。
  6. ページ名（例：レポート）を入力し「完了」。
  7. 「埋め込みコードを追加」をクリックし、作成したHTMLコードをすべて貼り付けて挿入する。
  8. 右上の「公開」ボタンを押し、適切な公開範囲（組織内のみ、など）を設定して公開。

【応用編】 GitHub Pages による公開

FTASやTIFOなどは、GitHub をベースにアプリを公開しています。

GitHub とは？

元々はソースコードを管理・共有するためのサービスですが、「Pages機能」を使うことで、Webサイトとして無料でホスティング（公開）することができます。

GitHub Pages のメリット

  • 画像の参照や複数のデータファイルの読み込みなど、複雑な構成のサイトも公開可能。
  • 完全無料で独自ドメインの設定も可能。
  • 更新履歴（バージョン）の管理が容易。
